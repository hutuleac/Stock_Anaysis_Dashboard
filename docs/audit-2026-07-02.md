# Platform Audit — 2026-07-02 (v0.15)

Scope: reliability + correctness of the full data pipeline, indicator math, scoring, setups, API layer. Tests: 187/187 pass. Build: clean. Live API checks run against both keys.

## Executive summary

The math layer (indicators.js, scoring.js, signals.js) is sound — formulas match their references and edge cases are guarded. The problems are in the **data feeding that math**: the weekly candles used by the setup engine are fabricated incorrectly on the TwelveData path (the only path that actually runs, since Finnhub free blocks candles), and two of the three market-regime inputs (VIX, short interest) silently return garbage/nothing on the free tier. The engine is correct; some of its inputs are not.

## Findings (ranked)

### F1 — CRITICAL: daily→weekly "resampling" is sampling, not aggregation
`App.svelte` (two places: refresh path ~line 220, hydrate path ~line 414) builds weekly candles by taking **every 5th daily bar's raw OHLCV** (`i % 5 === 0`). Real weekly bars need max(high), min(low), last(close), sum(volume) per week.
Consequences:
- Weekly ATR ≈ daily ATR (5× understated) → **EntryPanel's suggested stop (entry − 2×weekly ATR) is far too tight** — this directly harms the tool's core purpose.
- The last 1–4 trading days are **dropped entirely** (250 bars → indices 0,5,…,245), so all weekly setups (divergence, squeeze, structure, Setup Radar) run on data up to 4 days stale.
- Weekly volume dry-up/expansion runs on single-day samples — noise.
The Finnhub `W` path is correct but never executes (free tier 403s `/stock/candle`).
**Fix:** proper ISO-week aggregation of the daily candles, include the partial current week. ~15 lines, shared by both paths.

### F2 — HIGH: VIX is permanently 0, regime logic can never fire
Live check: `quote?symbol=VIX` returns all-zeros on Finnhub free; `^GSPC` returns an error body. `vixPrice = data.c ?? null` → **0, not null** (0 is not nullish). So "VIX > 25 → fundamentals 55%" and the regime notes are dead code in production, silently. MarketContextBar shows whatever `c=0` renders as.
Also: `fetchQuote('^GSPC')` result is never used → 1 wasted API call per refresh; `fetchQuote('VIX')` is a second wasted call.
No free real VIX source found (TwelveData 404s VIX/IXIC on free tier).
**Fix:** treat `c <= 0` as null everywhere vixPrice is derived; drop the two dead calls. Optionally derive a volatility regime from SPY realized volatility (20d stdev of returns, annualized) — computable from the SPY closes already fetched for RS, zero new calls.

### F3 — HIGH: short interest endpoint now 403 on this key
Live check: `/stock/short-interest` → 403. The v0.15 feature currently renders nothing (tombstone cache suppresses retries — good). Decide: remove the UI + call, or keep dormant.

### F4 — MEDIUM: "SPY downtrend" is actually "SPY down ≥0.5% today"
`spyDowntrend = spy.dp < -0.5` — a one-day move, not a trend. A single red day pulls every LONG score 20% toward 50; a green day in a real downtrend removes the penalty. SPY daily closes are already fetched (RS benchmark): use price vs SPY EMA50 (or 20>50 cross) instead. Zero new calls.

### F5 — LOW: CNN Fear & Greed is UA-gated (418 for bots)
Works from real browsers (verified: score 31.5 "Fear" with browser UA). Fine for the deployed app; will fail from scripts/some privacy proxies. Stale-cache fallback already handles it. No action needed, but the new dip card should degrade gracefully when F&G is null.

### F6 — LOW: rate-limiter race in `fetchMarketContext`
`Promise.all([fetchQuote(...), fetchQuote(...)])` — both read `_fhLastCall` before either writes it, so the 1.1s spacing is bypassed for that pair. Everywhere else calls are serialized. Disappears if F2's dead calls are removed.

### F7 — INFO: math verified correct
- RSI Wilder's, EMA (SMA seed), MACD, BB population stddev, ADX Wilder pipeline (post-v0.10 fix), Stoch: all match reference definitions.
- ATR simple-average divergence vs TradingView: documented, acceptable.
- signals.js detectors: guards correct, percentile gating as documented. One calibration note: the SQUEEZE phase uses absolute bandwidth <5%, which weekly stock data rarely hits — most squeeze signal comes via COMPRESSING (percentile-gated). Working as designed, just know ACT-via-squeeze is rare.
- radar.js gates and sort: correct; PEG-null passthrough is deliberate.
- News sentiment is naive keyword counting (known; 20% weight cap limits damage).

## Recommended fix order
1. F1 weekly aggregation (correctness of stops + all setups)
2. F2 VIX null-guard + remove 2 dead calls (+ optional SPY realized-vol regime)
3. F4 SPY trend from EMA, not one-day dp
4. F3 decision: remove or keep short interest UI

Then build the dip-entry card on the corrected base.
